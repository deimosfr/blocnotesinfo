1 Introduction

Sur des machines (surtout distantes), il est parfois pratique d'obtenir des informations du style, les espace de RAM disponible etc...

Bref, on peut avoir besoin d'autres informations, et je ne sais pas ou mettre tout ce petit monde.

2 Linux

2.1 dmesg

C'est ce qui est écrit lors du boot du kernel et tous les appels kernels (chargements de modules etc...) ça sera écrit ici. Sur certaines distribs, c'est même écrit dans les logs /var/log/*. Ce qui donne un truc du genre :

dmesg 
dmesg 
[    0.000000] Linux version 2.6.22-14-generic (buildd@king) (gcc version 4.1.3 20070929 (prerelease) (Ubuntu 4.1.2-16ubuntu2)) #1 SMP Tue Feb 12 02:46:46 UTC 2008 (Ubuntu 2.6.22-14.52-generic)
[    0.000000] Command line: root=/dev/mapper/lvm-racine ro quiet splash
[    0.000000] BIOS-provided physical RAM map:
[    0.000000]  BIOS-e820: 0000000000000000 - 000000000009fc00 (usable)
[    0.000000]  BIOS-e820: 00000000000f0000 - 0000000000100000 (reserved)
[    0.000000]  BIOS-e820: 0000000000100000 - 000000007fe0ac00 (usable)

2.3 Disques dur

Pour ce qui est des disques dur, utilisez la commande hdparm pour avoir toutes les infos :

hdparm 
$ hdparm -i /dev/sda
